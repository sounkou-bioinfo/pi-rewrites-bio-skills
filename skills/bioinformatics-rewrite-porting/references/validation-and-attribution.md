# Validation and Attribution

## Attribution

Make upstream credit visible in:

- README
- function or command docs
- citations
- release notes when relevant

## Validation

A compatibility claim should state:

- exact upstream version
- exact command used
- exact dataset used
- what equivalence means
  - byte identity
  - field equality
  - numeric tolerance

## Good practice

- prefer repo-stored validation scripts over undocumented manual comparison
- keep benchmark claims reproducible
- document unsupported features explicitly
